# CLAUDE.md — このプロジェクトでの Claude Code 動作指針

## プロジェクト概要

GROWTH-AI@AI活用ラボ のX（Twitter）アカウント運用を支援する自動化ワークフロー。
Grok API でリサーチし、Claude Code のスラッシュコマンドで投稿を生成・Typefully に送信する。

---

## 投稿作成時に必ず読み込むファイル

投稿を生成する際は、以下の順番でファイルを参照すること。

1. `00_ナレッジ/価値観.md` — 投稿の方向性・NG判定の最重要指針
2. `00_ナレッジ/最新コンテキスト.md` — 現在のフェーズ・優先テーマ
3. `00_ナレッジ/執筆スタイル.md` — 文体・構造・禁止ワード
4. `00_ナレッジ/投稿テンプレート/` — 使用するテンプレート型の個別ファイル

---

## キャラクター原則

**アカウント:** GROWTH-AI@AI活用ラボ  
**ポジション:** AIを使って実務・ビジネスを前進させる「実践型AIパートナー企業」  
**一言コンセプト:** 驚かせるのではなく、使えるAI情報で悩みを解決する

### 絶対にやらないこと

- 「これヤバい」「衝撃」「驚愕」「待って」などの煽り言葉
- 根拠のない誇張・未検証の未来予測
- ハッシュタグ（#）・絵文字（✨🔥など）の使用
- 「知らなかったら損」「乗り遅れる」などの恐怖マーケティング
- 他者・他社の批判・マウント

---

## 投稿フォーマット

- 1投稿：**140文字以内**
- スレッド形式：各ツイート140文字以内、最大3〜5ツイート
- 構造：① つかみ（15〜25文字）→ ② 本文・根拠（80〜100文字）→ ③ 締め（15〜25文字）
- 改行は意味のある区切りのみ。装飾的な改行はしない

---

## スラッシュコマンドと出力先

| コマンド | 処理内容 | 出力先 |
|---|---|---|
| `/research` | Grok で横断リサーチ実行 | `01_リサーチ/YYYY.MM.DD.md` |
| `/buzz-post <トピック>` | トピックから投稿生成 | `02_アウトプット/X投稿/下書き/` |
| `/x-post <URL>` | 海外ツイートを日本語スレッドに翻訳 | `02_アウトプット/X投稿/下書き/` |
| `/typefully-draft` | 下書きを Typefully に送信 | Typefully API |

## クローンからの使い方

### 1. リポジトリをクローン

```bash
git clone https://github.com/adachitoon/sns_ai.git
cd sns_ai
```

### 2. 依存パッケージをインストール

```bash
pip install openai python-dotenv
```

### 3. 環境変数を設定

```bash
cp .env.example .env
```

`.env` を開いて各APIキーを入力する。

```
GROK_API_KEY=取得したキーを貼り付け
TYPEFULLY_API_KEY=取得したキーを貼り付け
```

| 変数名 | 取得先 |
|---|---|
| `GROK_API_KEY` | https://console.x.ai |
| `TYPEFULLY_API_KEY` | Typefully → Settings → API |

### 4. Claude Code を起動

```bash
claude
```

### 5. 動作確認

```
/research
```

`01_リサーチ/YYYY.MM.DD.md` が生成されれば正常に動作している。

## 環境変数

`.env` ファイルに以下を設定（`.env.example` 参照）。

```
GROK_API_KEY=...       # Grok API（リサーチ・投稿生成）
TYPEFULLY_API_KEY=...  # Typefully API（下書き送信）
```

## 投稿生成前のチェックリスト

生成した投稿は以下をすべて通過してから出力すること。

- [ ] 根拠・事例・数字が含まれているか
- [ ] 禁止ワード・絵文字・ハッシュタグが含まれていないか
- [ ] 「なるほど」「試してみよう」を起点にしているか（煽っていないか）
- [ ] 会社として公開して恥ずかしくない内容か
