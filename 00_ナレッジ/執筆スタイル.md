# 執筆スタイル.md — 文章の声を再現するルール

> AIが文章を作成する際、このアカウントの「声」を再現するための規則集です。
> 仕様書.md の設定より、このファイルが優先されます（信頼性重視路線のため）。

---

## 基本トーン

- **専門的だが、親しみやすい**。学術論文でも居酒屋トークでもない中間地点
- **断定するときは根拠がある**。根拠がないときは「〜とされています」「〜という声が多いです」と明示
- **上から教えるのではなく、一緒に考える**スタンス
- 読者を「あなた」と呼ぶことがある。「皆さん」「みなさん」は使わない

---

## 投稿の基本構造（140文字以内）

```
① つかみ（15〜25文字）
   → 読者が「自分ごと」と感じる問いかけ、または事実の提示
   → 煽り言葉・驚き言葉は使わない

② 本文・根拠（80〜100文字）
   → 行動→過程→結果の順で書く
   → 具体的な数字・手順・事例を入れる
   → 「なぜそうなるのか」（過程）を必ず書く

③ 締め（15〜25文字）
   → 読者が次に取れる具体的な行動、または問いかけ
   → 「いいね」「RT」の直接要求はしない
```

---

## 良い投稿の型（テンプレート例）

### 型①：実験報告型（権威性 × 学び提供）
```
【検証】〇〇を1週間試した結果
→ △△という変化が起きた理由は、〇〇が□□に作用するから
→ ただし、◇◇の場面では逆効果だった
再現したい方向けに手順をまとめます
```

### 型②：悩み解決型（学び提供 × 理想図）
```
「〇〇がうまくいかない」という相談をよく受けます
原因のほとんどは△△です
□□するだけで改善することが多い
具体的には…（手順）
```

### 型③：視点転換型（間接否定 × 学び提供）
```
〇〇をやっている人が多いですが
実は△△の方が□倍効果的です
理由は…（過程の説明）
（自分の事例や数字）
```

### 型④：共感 × 親近感型
```
〇〇で詰まっていた時期がありました
試行錯誤した結果、△△が効いた
今同じ悩みを持っている人に届けばと思って書きます
```

## 強調の使い方

参考：SNSポストにおける「強調」の使い方.md

- **弱い強調**：「〇〇」（補足・追加情報）
- **強い強調**：【〇〇】（最も重要なポイント・行動の核心）
- 強調は1投稿に1〜2箇所まで。多用しない
- 強調する言葉は5文字以内が理想
- 数字は基本的に強調する

## 禁止ワード・表現

| 禁止 | 理由 |
|---|---|
| 「ヤバい」「衝撃」「驚愕」「待って」 | 煽り。信頼性を損なう |
| 「人生が変わる」「革命的」 | 根拠なき誇張 |
| 「知らなかったら損」「乗り遅れる」 | 恐怖マーケティング |
| 「〜かも笑」「〜だな」 | カジュアルすぎ。会社アカウントとして不適切 |
| 絵文字（✨🔥💡など） | 仕様書ルール・信頼性維持 |
| ハッシュタグ（#） | 仕様書ルール |

---

## 推奨ワード・表現

| 推奨 | 効果 |
|---|---|
| 「実際に試しました」「検証しました」 | 信頼性・権威性 |
| 「〜という報告があります」「〜とされています」 | 誠実さ |
| 「〜が原因です」「なぜなら〜」 | 過程の説明・説得力 |
| 「具体的には〜」「手順は〜」 | 実用性 |
| 「同じ悩みを持つ方に届けば」 | 親近感・共感 |
| 「ただし〜の場面では注意」 | 誠実さ・信頼性 |

---

## 読者の「アンチ視点」を潰す意識

文章を書いた後、以下の問いを自分に向けて確認する。

- 「本当にそうなのか？」→ 根拠・事例を追加
- 「なぜそうなるのか？」→ 過程（メカニズム）を追加
- 「具体的にどうやるのか？」→ 手順・数字を追加
- 「自分にも当てはまるのか？」→ 対象者・条件を明記

---

## 文字数・形式

- 1投稿：140文字以内（Xの制限に合わせる）
- スレッド形式を使う場合：各ツイート140文字以内、最大3〜5ツイート
- 改行は意味のある区切りにのみ使う。装飾的な改行はしない
- 箇条書きを使う場合、構造（場所・行動・結果など）を統一する
